Known problems to rectify next

- CustomerController is still large and mixes form validation, finance calculations, project creation, AJAX lookup responses, and mail/action helpers. It should be split later into smaller services or form requests.
- Customer create/edit Blade files still contain a large amount of duplicated JavaScript. The shared customer form behavior should be moved into a reusable partial or JS file.
- Some customer form JavaScript still contains typo text like "Select Loan Term</soption>" and "Select Loan Apr</soption>"; this should be corrected to proper option tags.
- Sub-contractor user loading is currently commented out in the customer edit view, so the sub-contractor user dropdown may not behave correctly when changing contractors.
- The customer form still depends on finance option ID 9 for Prepaid PPA behavior in JavaScript. The frontend should eventually use a backend flag or finance option property instead of a hard-coded ID.
- The controller checks Prepaid PPA by ID or name. This is safer than ID only, but a dedicated database flag would be cleaner.
- The ADU field is used to decide the starting subdepartment, but the validation rules do not explicitly validate ADU as yes/no. Add a nullable/in:0,1 or required/in:0,1 rule.
- Some optional fields such as sub_contractor_id, sub_contractor_user_id, notes, and sold_production_value are saved but do not yet have explicit validation rules.
- Customer update assumes the customer has related finance/project data in several places. More null-safe handling should be added for old or incomplete records.
- AJAX endpoints should eventually validate their request input before querying, especially IDs passed from dropdowns.
- Customer delete currently deletes project and customer, but related finance/adders/task cleanup should be reviewed to confirm database relationships or cascade deletes handle everything correctly.
- The customer create/edit pages should be manually tested end to end after validation errors, finance option changes, adder add/remove, ADU changes, and customer update.
